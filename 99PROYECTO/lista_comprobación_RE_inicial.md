# 🧩 Checklist – Entrevista inicial con el cliente

| Bloque | Pregunta / Aspecto a comprobar | ✔ / ~ / ✘ | Observaciones |
|--------|--------------------------------|-----------|---------------|
| **1. Negocio** | ¿A qué se dedica el cliente? | | |
| | ¿Qué problema quiere resolver? | | |
| | ¿Cómo trabaja actualmente? | | |
| | ¿Qué no funciona o quiere mejorar? | | |
| | ¿Quiénes son los usuarios del sistema? | | |
| | ¿Qué tareas realizan esos usuarios? | | |
| | ¿Qué tipo de información manejan? | | |
| **2. Funcionalidades** | ¿Qué quiere poder hacer el sistema? | | |
| | ¿Qué operaciones son imprescindibles? | | |
| | ¿Qué funcionalidades son opcionales? | | |
| | ¿Hay distintos tipos de usuarios? | | |
| | ¿Qué permisos tiene cada uno? | | |
| **3. Datos (muy importante)** | ¿Qué datos se van a almacenar? | | |
| | ¿Cómo se organizan actualmente esos datos? | | |
| | ¿En qué formato existen ahora (papel, Excel, etc.)? | | |
| | ¿Necesitan intercambiar datos con otros sistemas? | | |
| | ¿Deben estar estructurados los datos? | | |
| | ¿Se deben validar los datos? ¿Cómo? | | |
| **4. Interfaz / Web** | ¿Cómo quiere visualizar la información? | | |
| | ¿Necesita una página web? | | |
| | ¿Qué información se mostrará? | | |
| | ¿Debe ser accesible desde móvil? | | |
| | ¿Tiene ejemplos de páginas similares? | | |
| **5. Interacción (cliente)** | ¿El usuario puede modificar datos desde la web? | | |
| | ¿Habrá formularios? | | |
| | ¿Se necesitan acciones dinámicas sin recarga? | | |
| | ¿Se validan datos en el navegador? | | |
| **6. Intercambio de datos** | ¿Se importarán datos de otros sistemas? | | |
| | ¿Se exportarán datos? ¿En qué formato? | | |
| | ¿Se deben generar informes o documentos? | | |
| | ¿Se necesita transformar la información? | | |
| **7. Restricciones** | ¿Hay sistemas previos que mantener? | | |
| | ¿Funcionará en local o en internet? | | |
| | ¿Hay limitaciones de presupuesto o tiempo? | | |
| | ¿Debe funcionar sin conexión? | | |
| **8. Calidad / Pruebas** | ¿Qué ocurre si el sistema falla? | | |
| | ¿Qué errores son críticos? | | |
| | ¿Qué debe comprobarse antes de entregar? | | |
| | ¿Hay datos sensibles? | | |
| **9. Evolución** | ¿El sistema crecerá en el futuro? | | |
| | ¿Se añadirán nuevas funcionalidades? | | |
| | ¿Habrá más usuarios con el tiempo? | | |

# 🧠 Checklist – Dinámica de trabajo en la entrevista

| Aspecto | Pregunta / Indicador | ✔ / ~ / ✘ | Observaciones |
|--------|----------------------|-----------|---------------|
| **Organización previa** | ¿Han preparado la reunión antes de empezar? | | |
| | ¿Tienen claro qué quieren preguntar? | | |
| | ¿Se reparten roles (portavoz, anotador, etc.)? | | |
| **Gestión de la conversación** | ¿Hay un portavoz claro? | | |
| | ¿Evitan hablar todos a la vez? | | |
| | ¿Escuchan al cliente sin interrumpir? | | |
| | ¿Siguen un orden lógico en las preguntas? | | |
| **Toma de información** | ¿Alguien está tomando notas? | | |
| | ¿Recogen información relevante o solo superficial? | | |
| | ¿Piden aclaraciones cuando algo no queda claro? | | |
| **Participación** | ¿Participan varios miembros del equipo? | | |
| | ¿Hay alguien claramente desconectado? | | |
| | ¿Se apoyan entre ellos o se solapan? | | |
| **Actitud profesional** | ¿Mantienen un tono adecuado con el cliente? | | |
| | ¿Formulan preguntas claras y comprensibles? | | |
| | ¿Evitan lenguaje demasiado técnico? | | |
| **Capacidad de análisis** | ¿Profundizan en las respuestas o se quedan en lo superficial? | | |
| | ¿Relacionan unas preguntas con otras? | | |
| | ¿Detectan información importante por sí mismos? | | |
| **Cierre de la reunión** | ¿Hacen un pequeño resumen final? | | |
| | ¿Confirman que han entendido correctamente al cliente? | | |
| | ¿Detectan dudas pendientes? | | |
